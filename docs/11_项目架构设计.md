# 11 项目架构设计

## 1. 文档说明

本文档对 **Local RAG QA System** 的整体架构进行总结，包括系统架构、后端模块设计、RAG 数据流、项目目录及技术栈，帮助开发者快速理解整个项目。

---

# 2. 项目定位

本项目是一个基于 **llama.cpp + Qwen2-7B + FastAPI + LangChain + FAISS** 构建的本地 RAG（Retrieval-Augmented Generation）知识库问答系统。

项目特点：

- 本地部署大语言模型
- CPU 推理
- OpenAI Compatible API
- FastAPI 后端
- 多轮对话
- PDF 知识库
- RAG 检索增强
- 文件上传
- 工程化开发

整体目标：

> 构建一个完全本地运行、可持续扩展的 AI 问答平台。

# 4. 后端架构

项目后端采用 FastAPI 开发，整体目录如下：

```text
backend/
│
├── main.py
├── config.py
│
├── routers/
│   ├── chat.py
│   └── upload.py
│
├── services/
│   ├── llm.py
│   ├── history.py
│   ├── loader.py
│   ├── splitter.py
│   ├── embedding.py
│   ├── vector_store.py
│   ├── retriever.py
│   └── rag.py
│

│   └── system_prompt.txt
```

各模块职责如下：

| 模块 | 功能 |
|------|------|

| `config.py` | 项目配置 |
| `main.py` | FastAPI 启动入口 |

# 5. RAG 数据流

用户提问后，系统按照如下流程完成知识检索和回答生成：

```text
用户问题
      │
      ▼
FastAPI Chat API
      │
      ▼
Conversation History
      │
      ▼
Retriever
      │
      ▼
Embedding
      │
      ▼
FAISS Top-K
      │
      ▼
相关文档
      │
      ▼
Prompt Template
      │
      ▼
llama-server
      │
      ▼
Qwen2-7B
      │
      ▼
生成回答
```

整个流程全部在本地运行，无需访问外部模型服务。
