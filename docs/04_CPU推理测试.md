# 04 CPU 推理测试

## 1. 文档说明

本文档介绍如何使用 `llama.cpp` 进行本地 CPU 推理测试，验证模型能够正常运行，并说明常用推理参数的作用。

---

## 2. 启动模型

进入 `llama.cpp` 目录：

```bash
cd ~/llm_lab/llama.cpp
```

执行推理命令：

```bash
./build/bin/llama-cli \
    -m ../models/qwen2-7b-instruct-q4_k_m.gguf \
    -cnv \
    -t 8
```

---

## 3. 参数说明

| 参数 | 说明 |
|------|------|
| `-m` | 指定 GGUF 模型路径 |
| `-cnv` | 启用聊天（Conversation）模式 |
| `-t` | 设置 CPU 推理线程数 |

> **说明：**
>
> `-t` 参数建议根据 CPU 核心数量进行设置。例如，本项目使用 Intel Core i5-13420H（12 线程），实际测试采用 `-t 8`，能够兼顾推理性能和系统资源占用。

---

## 4. 推理测试

模型启动成功后，终端会进入交互模式：

```text
>
```

输入测试问题：

```text
你好
```

模型返回类似结果：

```text
你好！有什么可以帮助你的吗？
```

也可以继续输入其他问题，例如：

```text
请介绍一下 RAG。
```

模型能够连续回答，说明聊天模式运行正常。

---

## 5. 验证结果

若满足以下条件，则说明本地 CPU 推理成功：

- 模型能够正常加载
- 可以进入聊天模式
- 能够接收用户输入
- 能够生成完整回答
- 多轮对话运行正常

此时说明本地推理环境已配置完成。

---

## 6. 注意事项

- 首次加载模型时间较长，属于正常现象。
- CPU 推理速度受处理器性能和线程数影响，可根据实际情况调整 `-t` 参数。
- 如果模型无法启动，请检查：
  - 模型路径是否正确；
  - `LD_LIBRARY_PATH` 是否已配置；
  - GGUF 模型是否完整下载。

---

本章完成了 Qwen2-7B-Instruct 模型的本地 CPU 推理测试，验证了：

- `llama.cpp` 能够正常加载模型；
- CPU 推理环境配置正确；
- 模型支持交互式聊天；
- 为后续性能测试和 API 服务部署奠定基础。
