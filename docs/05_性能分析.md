# 05 Benchmark 性能分析

## 1. 测试命令

```bash
./build/bin/llama-bench \
-m models/qwen2-7b-instruct-q4_k_m.gguf \
-t 8
```

---

## 2. 测试结果

|Threads|Prompt(tok/s)|Generation(tok/s)|
|--------|------------|-----------------|
|1|9.66|3.09|
|2|18.87|5.45|
|4|29.06|7.64|
|6|30.36|8.81|
|8|32.39|9.43|
|12|34.38|8.19|

---

## 3. 分析

Prompt 阶段可以充分并行，因此随着线程数增加持续提升。

Generation 阶段属于自回归生成，需要逐 Token 推理，因此线程增加到一定程度后会受到 CPU Cache 和内存带宽限制。

本次测试中：

8 线程达到最佳生成速度。

---

## 4. 本章总结

最终采用：

```

-t 8

```

作为项目默认配置。
