# 05 Benchmark 性能分析

## 1. 文档说明

本文档使用 `llama.cpp` 自带的 `llama-bench` 工具，对 **Qwen2-7B-Instruct-Q4_K_M** 模型进行 CPU 推理性能测试，并根据测试结果选择项目默认线程配置。

---

## 2. 测试环境

| 项目 | 配置 |
|------|------|
| 操作系统 | WSL2 Ubuntu 24.04 |
| CPU | Intel Core i5-13420H |
| 推理框架 | llama.cpp |
| 模型 | Qwen2-7B-Instruct-Q4_K_M |
| 推理方式 | CPU |
| 测试工具 | llama-bench |

---

## 3. 测试命令

进入 `llama.cpp` 目录：

```bash
cd ~/llm_lab/llama.cpp
```

执行 Benchmark：

```bash
./build/bin/llama-bench \
    -m ../models/qwen2-7b-instruct-q4_k_m.gguf \
    -t 8
```

> **说明：**
>
> `-t` 用于指定 CPU 推理线程数。为了寻找最佳性能，本项目分别测试了多个线程配置。

---

## 4. 测试结果

| Threads | Prompt (tok/s) | Generation (tok/s) |
|---------:|---------------:|-------------------:|
| 1 | 9.66 | 3.09 |
| 2 | 18.87 | 5.45 |
| 4 | 29.06 | 7.64 |
| 6 | 30.36 | 8.81 |
| 8 | **32.39** | **9.43** |
| 12 | 34.38 | 8.19 |

---

## 5. 性能分析

### Prompt 阶段

Prompt（提示词处理）属于并行计算任务，能够充分利用多线程，因此随着线程数增加，处理速度持续提升。

### Generation 阶段

Generation（文本生成）采用自回归（Autoregressive）方式逐 Token 推理，每生成一个 Token 都依赖前一个 Token，因此并行能力有限。

随着线程数继续增加，会受到以下因素影响：

- CPU Cache 命中率下降；
- 内存带宽成为瓶颈；
- 线程调度开销增加。

因此，生成速度不会随着线程数无限提升。

---

## 6. 测试结论

综合 Prompt 和 Generation 两项指标：

- Prompt 性能随线程数增加持续提升；
- Generation 在 **8 线程**时达到最佳性能；
- 超过 8 线程后，生成速度反而略有下降。

因此，本项目最终采用：

```bash
-t 8
```

作为默认线程配置。
