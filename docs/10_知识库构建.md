# 10 知识库构建

## 1. 文档说明

本文档介绍本项目知识库的构建流程，包括 PDF 文档导入、向量索引生成、知识库更新以及动态扩展机制。

知识库是 RAG 系统的重要组成部分，通过构建向量数据库，使大语言模型能够结合外部文档进行回答。

---

## 2. 知识库目录

本项目采用如下目录组织知识库：

```text
data/
├── pdf/
│   ├── document1.pdf
│   ├── document2.pdf
│   └── ...
└── faiss/
    ├── index.faiss
    └── index.pkl
```

各目录作用如下：

| 目录 | 作用 |
|------|------|
| `data/pdf/` | 存放原始 PDF 文档 |
| `data/faiss/` | 存放 FAISS 向量索引 |

其中：

- `pdf` 保存原始知识库；
- `faiss` 保存构建完成的向量数据库。

---

## 3. 知识库构建流程

整个知识库构建流程如下：

```text
上传 PDF
      │
      ▼
保存到 data/pdf
      │
      ▼
读取 PDF
      │
      ▼
文本切分
      │
      ▼
Embedding
      │
      ▼
建立 FAISS Index
      │
      ▼
保存 data/faiss
```

索引建立完成后，即可用于后续 RAG 检索。

---

## 4. PDF 上传

项目提供上传接口：

```http
POST /upload
```

支持上传：

```text
application/pdf
```

上传后的文件统一保存到：

```text
data/pdf/
```

便于后续统一管理。

---

## 5. 重建向量索引

当新增或修改 PDF 后，需要重新构建向量数据库。

执行：

```bash
python scripts/build_index.py
```

脚本执行流程：

```text
扫描 data/pdf/

↓

读取 PDF

↓

Text Splitter

↓

Embedding

↓

FAISS

↓

保存 data/faiss/
```

完成后即可加载最新知识库。

---

## 6. 知识库更新流程

新增知识时，无需重新训练模型。

更新流程如下：

```text
新增 PDF

↓

上传知识库

↓

重建 FAISS

↓

重新启动服务（或重新加载索引）

↓

即可检索最新内容
```

整个过程无需修改模型参数，仅更新向量数据库即可。

## 8. 项目优势

本项目知识库具有以下特点：

- 支持动态上传 PDF
- 支持本地知识管理
- 无需重新训练模型
- 基于 FAISS 实现高效检索
- 可扩展更多文档格式（Word、Markdown、TXT 等）
- 支持持续扩展知识库
