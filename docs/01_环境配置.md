# 01 环境配置

## 1. 文档说明

本文档介绍项目开发环境的搭建过程，包括 WSL2、Ubuntu 以及项目开发所需工具的安装，为后续编译 `llama.cpp` 和部署 Qwen2 模型做好准备。

---

## 2. 项目简介

本项目基于 **llama.cpp** 在 **WSL2 Ubuntu** 环境下部署 **Qwen2-7B-Instruct GGUF**，实现本地 CPU 推理，并进一步构建基于 **FastAPI + LangChain + FAISS** 的本地 RAG（Retrieval-Augmented Generation）知识库问答系统。

项目目标包括：

- 本地部署大语言模型
- CPU 推理
- OpenAI Compatible API
- FastAPI 后端
- RAG 知识库
- PDF 文档检索
- 多轮对话

---

## 3. 开发环境

| 软件 | 版本 |
|------|------|
| Windows | Windows 11 |
| WSL | WSL2 |
| Ubuntu | Ubuntu 24.04 LTS |
| GCC | 系统默认版本 |
| CMake | 系统默认版本 |
| Git | 系统默认版本 |

> **说明：**
> Ubuntu、GCC、CMake 等版本无需完全一致，只要满足 `llama.cpp` 官方要求即可。

---

## 4. 安装开发工具

更新软件源：

```bash
sudo apt update
```

安装项目依赖：

```bash
sudo apt install -y \
build-essential \
cmake \
git
```

安装完成后验证版本：

```bash
gcc --version
g++ --version
cmake --version
git --version
```

示例输出：

```text
gcc (Ubuntu 13.x.x)

cmake version 3.xx.x

git version 2.xx.x
```

---

## 5. 项目目录

建议采用如下目录结构：

```text
~/llm_lab/
├── llama.cpp/
│   ├── build/
│   ├── docs/
│   ├── scripts/
│   ├── src/
│   ├── include/
│   ├── CMakeLists.txt
│   └── ...
└── models/
    └── qwen2-7b-instruct-q4_k_m.gguf
```

其中：

| 目录 | 作用 |
|------|------|
| llama.cpp | llama.cpp 源码及编译结果 |
| build | 编译输出目录 |
| docs | 项目开发文档 |
| scripts | 启动脚本 |
| models | GGUF 模型文件 |

---

## 6. 验证环境

执行以下命令确认环境正常：

```bash
git --version

cmake --version

gcc --version
```

如果均能正常输出版本信息，则说明开发环境配置完成。
